# Bug Triage Policy

**Severity**
- S1: System down / data loss / wrong totals/payments
- S2: Major flow broken / no workaround
- S3: Minor defect / workaround exists
- S4: Cosmetic/content

**Priority**
- P0: Immediate fix
- P1: Next release
- P2: Backlog

**SLAs**
- S1/P0: 4h fix window
- S2/P1: 1 business day
- S3/P2: Next sprint
